Here's the work flow:

I'm gonna show you what the current EE301 microcontroller lab is like (two powerpoints), and we are going to remake them. The format will be based on the main.tex file attached, but we'll replace all the actual content with the new lab we are making. The new lab should follow the workflow



0) Download and install Visual Studio Code on their individual laptops. Let the installation run in the background while complete the next step.

1) Instruct students to solder the male header onto the PCB (see https://usma-eecs.github.io/CY105_Lab/Initial_Steps.html, "Assembly"). Include necessary tips as well as required equipment (eye protection, ventilating fan, soldering iron set to 750F, solder, with a mild warning that solder contains lead and they must use a pair of pliers to hold up the solder or wash their hands afterwards). Include instructional graphics as appropriate. Note that the soldering should happen on the "Beat Navy!" side of the PCB. Include examples of good and bad solder joints from the link above.

2) Once the soldering is complete and satisfactory, install the adafruit circuit playground Bluefruit onto the PCB using nuts and bolts. Note that they need a screwdriver, and a pair of pliers to make sure the bolts are tightened (but not overly so). Should be snug with no play. Note to them that the bluefruit should be installed on the "Beat Navy!" side as well.

3) Once complete, power on the Bluefruit by plugging it into their laptops using the male-male USB-A/micro USB-B cable. 

4) Download the "Bluefruit Reflash Kit v2026.3" from Canvas, and decompress the zip file. 

5) Double click the small reset button in the center of bluefruit to put it into bootloader mode. All LEDs, known as neopixels, should be green.

6) Run the "autoflash" file and follow its instructions. (File attached. We'll talk about updating it in a sec, but plan to include what the prompts should be like.) Neopixels should turn to all red while bootloader is running.
